Bitmap Fuzzing Project

Welcome to the fuzz the bitmap file!

File format fuzzing is a testing method that identifies vulnerabilities by providing unexpected or malformed files to applications. This technique helps uncover bugs, such as crashes or security flaws, in software handling various file types.

Bitmap fuzzing is a technique used to test software for vulnerabilities by generating and feeding random or malformed bitmap image files. This process helps reveal issues like crashes, buffer overflows, or memory leaks in applications that handle bitmap images, enhancing software robustness and security.

The sample bitmap fuzzing done in this terminal

Step 1 : Type ls command in the terminal and click run button. The list of files will appear.

Step 2 : Type pwd command in the terminal and click run button. It will show the current directory.

Step 3 : The sample bitmap file is stored in the name called sam.bmp and the fuzz target is in the file named fuzz_target.c These are the files needed for fuzzing.

Step 4 : Type ./fuzz_target public/inputs/sam.bmp in the terminal and click run button. The libfuzzer will fuzz the bitmap file and give the output.

LibFuzzer

LibFuzzer is a powerful tool for fuzz testing, integrated with LLVM. It efficiently detects vulnerabilities in your bitmap processing code by generating random inputs and monitoring for crashes.

  • It works by mutating inputs and monitoring the execution path to detect bugs and vulnerabilities.
  • Targeted at C/C++ applications, especially those with functions that process user inputs.
  • Integrates directly with LLVM-based projects and requires compilation with Clang.
  • Efficient in detecting memory errors like buffer overflows.
  • Supports integration with sanitizers like AddressSanitizer and MemorySanitizer for more accurate results.

AFL (American Fuzzy Lop)

AFL is a widely used fuzz testing tool that employs genetic algorithms to find bugs in applications. It automatically generates test cases based on the program's feedback.

  • It is primarily used for fuzzing C/C++ applications but can be adapted for other languages.
  • AFL requires instrumentation of the target program to track code coverage during execution.
  • It generates random inputs and evolves them based on how much code they cover, improving fuzzing efficiency.
  • AFL helps find memory corruption bugs such as buffer overflows

LibFuzzer vs. AFL

Feature LibFuzzer AFL
Integration Integrates with LLVM Standalone tool
Performance Optimized for speed Good performance, but slower than LibFuzzer
Input Generation Generates inputs based on code coverage Uses genetic algorithms for input generation
Setup Requires LLVM setup Easy to set up

Applications of Bitmap Fuzzing

Bitmap fuzzing is widely applicable in various fields, including:

  • Image Processing Software: Tests for vulnerabilities in image rendering and processing.
  • Data Conversion Tools: Ensures safe conversion between bitmap and other file formats.
  • Game Development: Validates and secures the handling of image assets in games.
  • Operating Systems & File Viewers: Identifies potential vulnerabilities in systems displaying bitmap files.
  • Cybersecurity Defense: Helps defend against maliciously crafted bitmap files by finding vulnerabilities.
  • Digital Forensics: Ensures forensic tools can handle malformed bitmap files without data loss.
